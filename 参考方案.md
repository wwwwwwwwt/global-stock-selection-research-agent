 股市监控 Agent 设计方案

## 1. 项目概述

一个面向美股与中国股市的混合型量化监控 Agent，支持三层监控体系（固定股票池跟踪、全市场扫描、事件驱动分析），结合技术面/基本面/宏观多维度策略，并通过 LLM 生成每日摘要见解。

**目标用户**：个人投研为主，可扩展至小团队共享。
**数据来源**：可插拔设计，初期使用免费数据源（Yahoo Finance、AKShare）。
**交付形式**：Web Dashboard + 多通道推送（钉钉/飞书/邮件/企业微信）+ 每日定时报告。

## 2. 总体架构

### 2.1 模块划分

系统采用分层模块化单体架构，内部按层严格解耦，未来可独立拆分为微服务：

```
┌─────────────────────────────────────────────────────────┐
│                     API Layer (FastAPI)                  │
│  ┌─────────┐ ┌─────────┐ ┌─────────┐ ┌───────────────┐ │
│  │ 股票池  │ │ 扫描器  │ │ 事件流  │ │   LLM摘要    │ │
│  │  接口   │ │  接口   │ │  接口   │ │    接口      │ │
│  └────┬────┘ └────┬────┘ └────┬────┘ └───────┬───────┘ │
└───────┼───────────┼───────────┼──────────────┼─────────┘
        │           │           │              │
┌───────▼───────────▼───────────▼──────────────▼─────────┐
│              Service Orchestration Layer               │
│         (定时调度 APScheduler + 事件分发器)              │
└───────┬───────────┬───────────┬────────────────────────┘
        │           │           │
┌───────▼───┐ ┌────▼────┐ ┌────▼────┐ ┌───────────────┐
│ Data Feed │ │ Strategy│ │Predictor│ │ Notification  │
│  Layer    │ │ Engine  │ │ Service │ │    Layer      │
│(可插拔源)  │ │(策略注册) │ │(模型管理) │ │ (多通道推送)  │
└───────────┘ └─────────┘ └─────────┘ └───────────────┘
        │                              │
        └──────────────┬───────────────┘
                       │
            ┌──────────▼──────────┐
            │   Storage Layer     │
            │ (PostgreSQL + Redis)│
            └─────────────────────┘
```

### 2.2 核心数据流

**流 1：日常跟踪（定时）**
```
APScheduler 触发 → Data Feed 拉取股票池 OHLCV → Strategy Engine 运行核心策略
→ Predictor Service 运行价格预测 → 结果写入 DB → LLM 层生成摘要 → Notification 推送
```

**流 2：全市场扫描（定时）**
```
APScheduler 触发 → Data Feed 批量拉取 300 只股票数据 → Strategy Engine 运行筛选策略
→ 命中标的进入深度分析通道 → 结果写入 DB → 前端 Dashboard 展示
```

**流 3：事件驱动（异步）**
```
外部事件源（财报日历、宏观日程）→ 事件分发器识别事件类型 → 触发对应分析 Pipeline
→ Data Feed 补充事件相关数据 → Strategy Engine 做事件影响分析 → 即时推送告警
```

## 3. 数据采集层（Data Feed Layer）

### 3.1 适配器模式

所有数据源通过统一接口接入：

```python
class BaseDataFeed(ABC):
    @abstractmethod
    def fetch_ohlcv(self, symbol: str, start: date, end: date, interval: str) -> pd.DataFrame: ...

    @abstractmethod
    def fetch_fundamentals(self, symbol: str) -> dict: ...

    @abstractmethod
    def fetch_macro(self, indicators: list[str], date: date) -> dict: ...
```

### 3.2 初期适配器实现

| 适配器 | 覆盖市场 | 数据类型 | 限制 |
|--------|---------|---------|------|
| `YahooFinanceFeed` | 美股、港股、部分中概 | OHLCV、基本面 | 免费，有频率限制 |
| `AKShareFeed` | A股、宏观 | OHLCV、财报、宏观 | 免费，国内数据源 |

配置项：`DATA_FEED_PROVIDER=yahoo` 或 `akshare`。

## 4. 策略引擎层（Strategy Engine）

策略按三类注册到统一的策略注册表，所有策略实现统一接口：

```python
class BaseStrategy(ABC):
    @abstractmethod
    def analyze(self, symbol: str, data_bundle: DataBundle) -> StrategyResult: ...
```

`DataBundle` 包含该 symbol 的 OHLCV、基本面、宏观数据，策略之间通过 DataBundle 解耦。

### 4.1 核心策略（Core Strategies）

- `MovingAverageCrossStrategy` — 均线交叉（ golden cross / death cross ）
- `MomentumStrategy` — RSI / MACD 动量信号
- `VolatilityBreakoutStrategy` — 布林带/ATR 波动率突破
- `SupportResistanceStrategy` — 支撑位/阻力位识别

### 4.2 预测策略（Prediction Strategies）

- `LSTMPricePredictor` — 短期价格时序预测
- `TrendForecastStrategy` — 中期趋势方向预测（机器学习分类）

### 4.3 回归策略（Regression Strategies）

- `FactorRegression` — 多因子收益归因（市值、估值、动量、质量因子）
- `MacroRegression` — 股价与宏观变量（利率、汇率、PMI）回归分析
- `FundamentalRegression` — PE/PB-ROE 回归、营收增速-股价弹性

## 5. 预测服务层（Predictor Service）

### 5.1 模型生命周期

- **训练**：定时在后台训练/更新模型（LSTM、XGBoost、线性回归等），模型版本化存储
- **推理**：策略调用时加载对应模型版本做推理
- **回测**：每个预测模型自带回测接口，评估近 N 天预测准确率

### 5.2 模型存储

- 模型文件：本地文件系统 `models/{symbol}/{model_name}_v{version}.pkl`
- 模型元数据（版本、训练时间、回测指标）：PostgreSQL

## 6. 任务编排层（Service Orchestration）

### 6.1 双驱动设计

| 驱动类型 | 触发源 | 用途 |
|---------|--------|------|
| 定时驱动 | APScheduler | 每日收盘后跑股票池分析、每小时扫描市场、定时训练模型 |
| 事件驱动 | 事件分发器 | 财报发布、宏观数据公布、价格异动（涨跌幅超过阈值） |

### 6.2 事件分发器

```python
class EventDispatcher:
    def dispatch(self, event: MarketEvent):
        handlers = self.registry.get_handlers(event.type)
        for handler in handlers:
            handler.process(event)
```

支持的事件类型：`EARNINGS_RELEASE`、`MACRO_DATA`、`PRICE_SPIKE`、`VOLUME_ANOMALY`

## 7. API 层（FastAPI）

核心端点：

- `GET /api/portfolio` — 股票池状态与策略信号
- `GET /api/scan` — 全市场扫描结果
- `GET /api/events` — 事件流与影响分析
- `POST /api/query` — 自然语言查询（对接 LLM 摘要层）
- `GET /api/reports/daily` — 每日摘要报告

## 8. 前端（React + Recharts）

页面规划：

- **Dashboard** — 股票池概览、策略信号面板、收益归因图表
- **Scanner** — 全市场扫描结果列表，支持按策略信号过滤
- **Event Stream** — 事件时间线，带影响分析
- **Report** — 每日摘要报告展示

## 9. 推送层（Notification Layer）

### 9.1 多通道适配器

- `DingTalkNotifier`
- `FeishuNotifier`
- `EmailNotifier`
- `WeComNotifier`

### 9.2 推送分级

| 级别 | 场景 | 方式 |
|------|------|------|
| 紧急 | 价格异动、重大事件 | 即时推送 |
| 日常 | 每日收盘摘要 | 定时推送 |
| 周度 | 组合回顾与下周展望 | 每周推送 |

## 10. LLM 摘要层

- **输入**：当日所有策略结果、预测输出、事件影响、市场数据
- **处理**：构造结构化 Prompt，调用 Claude / OpenAI API
- **输出**：自然语言摘要，分"市场概览"、"重点关注"、"风险提示"三部分

## 11. 存储层

| 数据类型 | 存储 | 说明 |
|---------|------|------|
| OHLCV / 基本面 / 宏观数据 | PostgreSQL | 时序数据，按 symbol + date 分区 |
| 策略结果 / 事件 / 报告 | PostgreSQL | 结构化数据 |
| 任务队列 / 缓存 | Redis | APScheduler 作业状态、热点数据缓存 |
| 模型文件 | 本地文件系统 | `models/{symbol}/{model_name}_v{version}.pkl` |

## 12. 错误处理策略

| 场景 | 处理方式 |
|------|---------|
| 数据采集失败 | 自动重试 3 次（指数退避），失败后记录到 `failed_fetches` 表 |
| 策略执行异常 | 单策略失败不影响其他策略，错误隔离 |
| 模型推理失败 | 回退到规则基准（如均线策略直接用最新均线值） |
| LLM API 失败 | 缓存上一次有效摘要，标记"摘要生成失败" |

## 13. 部署方案

- **开发环境**：Docker Compose（PostgreSQL + Redis + App）
- **生产环境**：单节点 Docker 部署，未来若流量大可将 Predictor Service 独立拆出

## 14. 技术栈

| 层次 | 技术 |
|------|------|
| 后端框架 | Python + FastAPI |
| 任务调度 | APScheduler + 自定义事件分发器 |
| 数据库 | PostgreSQL + Redis |
| 前端 | React + Recharts |
| ML/预测 | PyTorch (LSTM) + XGBoost + scikit-learn |
| LLM | Claude API / OpenAI API |
| 部署 | Docker + Docker Compose |

## 15. 监控范围

- **美股/中概**：扫描 100 只（Yahoo Finance）
- **A股**：扫描 100 只（AKShare）
- **固定股票池**：用户自选股（数量不限，日常跟踪）
- **宏观指标**：美联储利率、CPI、中国 PMI、汇率等
