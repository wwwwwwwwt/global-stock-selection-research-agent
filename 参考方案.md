# 全球市场感知量化选股研究系统设计方案

## 1. 项目定位

OpenStockAgent 的目标不是预测某一只股票未来几根 K 线，而是从股票宇宙中筛出值得关注的候选股，并给出可追溯的入选理由、风险提示、观察条件和失效条件。

系统采用“自下而上 + 自上而下”的研究框架：

- 自下而上：个股 K 线、量价、技术因子、缠论结构、基本面、估值、可选 Kronos 因子。
- 自上而下：全球指数、波动率、汇率、利率、商品、行业轮动、主题热度、国际新闻和宏观事件。
- 解释层：LLM 只负责基于结构化证据解释候选股，不直接拍脑袋选股。

核心流程：

```text
Universe -> Data -> Factors -> Market Context -> Ranking -> Evidence -> LLM Explanation
```

## 3. 股票宇宙层

股票宇宙定义“从哪里选”。

初期支持：

- A 股样本池。
- 港股样本池。
- 美股样本池。
- 用户自选池。
- 主题池，例如 AI、半导体、能源、黄金、防务。

核心表：

```text
universes
  universe_id, name, market, asset_type, description, created_at

universe_members
  universe_id, instrument_id, start_date, end_date, reason
```

所有选股任务都必须指定 `universe_id`，避免候选范围隐式变化。

## 4. 标准数据层

标准数据层负责把不同来源的数据统一成内部格式。策略、因子、缠论、LLM 都不能直接依赖 Yahoo、AKShare 或其他外部源。

核心表：

```text
instruments
instrument_aliases
bars
feed_runs
data_quality_issues
prediction_runs
predicted_bars
```

`bars` 只存事实型 K 线数据，不存金叉、缠论结构、情绪分、选股分或 LLM 文本。

K 线字段：

```text
instrument_id
timestamp
local_date
interval
source
adjustment
open
high
low
close
volume
amount
currency
is_complete
```

## 5. 因子层

因子层负责把个股证据转成可横截面比较的数值。

初期因子：

```text
return_5d
return_20d
return_60d
ma_trend_score
ma_slope_20d
volume_expansion_20d
atr_14d
max_drawdown_20d
turnover_amount_20d
```

后续扩展：

```text
value        PE, PB, 股息率
quality      ROE, 毛利率, 现金流质量, 杠杆
growth       收入增速, 利润增速
sentiment    新闻情绪, 事件语气
theme        AI, 半导体, 能源, 黄金, 防务等主题暴露
theory       缠论结构分, 买卖点类型, 背驰强度
kronos       方向分, 置信度, 预测波动
```

核心表：

```text
factor_definitions
factor_values
```

每个因子必须记录版本、方向、分位数、z-score 和证据 JSON。

## 6. 经典理论层

缠论等经典理论作为结构识别引擎接入，不直接替代选股排序。

缠论 MVP：

```text
包含关系处理
顶/底分型
基础笔
简化中枢
基础背驰
二买/二卖候选信号
```

核心表：

```text
theory_structures
  structure_id, instrument_id, interval, theory_name,
  structure_type, direction, start_timestamp, end_timestamp,
  strength, confidence, evidence_json, version

theory_signals
  signal_id, instrument_id, interval, theory_name,
  signal_type, direction, strength, confidence,
  summary, evidence_json, version
```

缠论输出是选股证据，不是交易指令。

## 7. 全球市场上下文层

市场上下文层负责捕捉整体环境和国际新闻对选股的影响。

输入：

```text
SPX, NDX, DJI, HSI, CSI300, Nikkei
VIX
DXY, USDCNH, USDJPY
US10Y, Fed policy expectations
gold, oil, copper
geopolitics, tariffs, central banks, inflation, industry policy
```

核心表：

```text
global_market_bars
market_context_snapshots
news_documents
market_events
theme_scores
instrument_theme_exposures
```

市场上下文输出：

```text
risk_regime               risk_on / neutral / risk_off
liquidity_score
volatility_score
usd_strength_score
commodity_pressure_score
sector_rotation_json
theme_heat_score
news_event_adjustment
```

国际新闻不会直接选股，而是先结构化为事件、主题和行业影响，再修正候选股分数。

## 8. 筛选与排序层

筛选分两步。

硬过滤：

```text
成交额过低
ST 或停牌
上市时间过短
近期 K 线缺失
严重数据质量问题
财务风险过滤
黑名单或暂不支持市场
```

打分公式：

```text
total_score =
  individual_factor_score
+ theory_structure_score
+ sector_rotation_score
+ theme_alignment_score
+ market_regime_adjustment
+ news_event_adjustment
+ kronos_optional_score
- risk_penalty
```

核心表：

```text
screen_strategies
screen_runs
screen_results
```

候选结果必须包含：

```text
rank
total_score
score_breakdown_json
reason_json
risk_json
evidence_refs_json
```

## 9. LLM 解释层

LLM 不负责直接选股。LLM 只消费结构化证据并生成研究解释。

输入：

```text
候选股排名和分数
因子贡献
缠论或其他理论结构
全球市场上下文
相关新闻事件
数据质量备注
确认条件
失效条件
```

输出：

```text
为什么入选
主要支持证据
主要风险
市场环境影响
后续观察条件
失效条件
```

所有 LLM 结论必须能追溯到 `evidence_refs_json`。

## 10. 阶段规划

### Phase 0：产品重定位

- README 和项目说明从单股预测切换为量化选股。
- Kronos 改为可选因子来源。

### Phase 1：标准数据底座

- 实现 `instruments`、`aliases`、`bars`、`feed_runs`、`data_quality_issues`。
- 增加 `CsvFeed`，确保测试不依赖外网。
- Yahoo 迁移为标准 `fetch_bars`。

### Phase 2：股票宇宙

- 实现 `universes` 和 `universe_members`。
- 支持 CSV 样本池和自选池。

### Phase 3：因子引擎

- 实现动量、趋势、量价、波动因子。
- 计算分位数和 z-score。

### Phase 4：筛选排序

- 实现硬过滤、加权打分、Top N 输出。
- 生成 `screen_runs` 和 `screen_results`。

### Phase 5：市场上下文

- 实现全球资产样本数据。
- 生成 `market_context_snapshots`。
- 初步判断 risk_on / neutral / risk_off。

### Phase 6：新闻事件和主题

- 先用 CSV 或手动样本接入新闻。
- 结构化为事件、主题、行业影响。

### Phase 7：缠论 MVP

- 实现分型、笔、简化中枢和二买/二卖候选。

### Phase 8：LLM 证据上下文

- 生成选股解释上下文。
- 初期先不调用真实 LLM。

### Phase 9：回测评估

- 评估 Top N 组合未来收益、回撤、命中率和换手。
- 按市场状态分组评估。

### Phase 10：产品界面

- 先做 CLI 和日报。
- 再考虑 Dashboard。

## 11. 当前优先级

立即优先级：

1. 完成标准数据底座。
2. 增加股票宇宙。
3. 实现第一批因子。
4. 做出第一个可解释的候选股排名。

在这四步完成之前，不引入复杂前端、自动交易、实时流或完整 LLM 自动分析。
